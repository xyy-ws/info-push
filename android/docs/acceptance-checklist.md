# Info Push Android 验收清单

> 工作目录：`apps/info-push/android`

## 0. 构建与基础检查

- [ ] 执行单测：`./gradlew --no-daemon :app:testDebugUnitTest`
- [ ] 执行构建：`./gradlew --no-daemon :app:assembleDebug`
- [ ] Debug APK 产物存在：`app/build/outputs/apk/debug/app-debug.apk`
- [ ] 应用可正常启动，导航链路可用（首页 → 信息源 → 收藏 → 消息中心 → 设置）

---

## 1) 信息源管理（Sources）

### 1.1 本地信息源
- [ ] 进入“信息源”页，能看到信息源列表
- [ ] 新增信息源（名称/URL/类型/标签）成功
- [ ] 启用/禁用切换生效
- [ ] 删除信息源生效
- [ ] 本地筛选（名称/URL/标签）生效

### 1.2 状态反馈
- [ ] loading 态显示统一文案（加载中）
- [ ] empty 态文案清晰（无数据/筛选无结果）
- [ ] error 态文案清晰，失败时可见

---

## 2) AI 搜索

- [ ] 输入关键词后点击“AI 搜索信息源”可发起请求
- [ ] loading 态可见（AI 搜索中）
- [ ] 无结果时出现 empty 提示
- [ ] 错误时出现 error 提示
- [ ] 搜索结果支持“打开链接”
- [ ] “一键添加”后按钮状态变为“已添加”（避免重复添加）

---

## 3) 首页切源 & 刷新（Feed）

- [ ] 首页显示信息源切换入口（按钮行）
- [ ] 切换信息源后，当前源名称更新
- [ ] 点击“刷新当前信息源”可触发刷新
- [ ] 点击“同步远端首页”可触发拉取
- [ ] loading / empty / error 态展示正常
- [ ] 列表项可“打开原文”

---

## 4) 收藏（Favorites）

- [ ] 收藏页接入统一状态反馈组件（loading / empty / error）
- [ ] 收藏列表项卡片样式、间距、层级显示正常
- [ ] 列表项可“打开原文”
- [ ] “同步收藏”为主按钮，“去消息中心”为次按钮
- [ ] mock 回退时可见提示（当前显示 mock 数据）

---

## 5) 消息（Messages）

- [ ] 消息页接入统一状态反馈组件（loading / empty / error）
- [ ] 消息列表项卡片样式、间距、层级显示正常
- [ ] 可从标题/正文提取链接并“打开链接”
- [ ] “同步消息”为主按钮，“去设置”为次按钮
- [ ] mock 回退时可见提示（当前显示 mock 数据）

---

## 6) 导入导出（Settings）

- [ ] 设置页存在导出入口
- [ ] 设置页存在导入入口（replace / merge）
- [ ] 导出 JSON 包含 `version / exportedAt / data`
- [ ] replace 导入后本地数据按全量替换预期更新
- [ ] merge 导入后本地数据按合并策略更新

---

## 7) 链接打开能力

- [ ] Feed 列表“打开原文”可调用外部浏览器
- [ ] Sources 中 URL “打开链接”可调用外部浏览器
- [ ] Favorites “打开原文”可调用外部浏览器
- [ ] Messages “打开链接”可调用外部浏览器
- [ ] 非法链接不会导致崩溃（有失败兜底）

---

## 8) 回归建议（轻量）

- [ ] 快速验证四个主页面在 loading / empty / error 三态下 UI 不错位
- [ ] 检查按钮主次关系一致：主操作 `Button`，次操作 `OutlinedButton`
- [ ] 检查列表项间距统一、无明显拥挤/断层
