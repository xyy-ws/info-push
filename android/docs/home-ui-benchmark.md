# 首页 Feed 快速参考调研（2026-02）

目标：提炼 Google News / Apple News / Feedly / SmartNews / Reddit / X 等主流信息流首页中，可直接落地到当前 Android 首页首屏的模式。

## 共性模式（可落地）

1. **顶部信息架构三段式**
   - 一级：页面标题（Home/For you）
   - 二级：当前上下文（当前源/分组）
   - 三级：轻量操作（刷新、同步、更多）
   - 落地：将“标题 + 当前源 + 操作按钮”合并为同一顶部容器，降低页面碎片感。

2. **源切换产品化控件（可滚动 chips/segmented）**
   - 主流产品都避免纯文本按钮堆叠，使用可横滑标签、选中态强反馈。
   - 落地：信息源改为横向 `FilterChip`，选中态带 icon 与视觉差异。

3. **卡片内容层级明确**
   - 标题 > 摘要 > 元信息（时间/来源） > 行动（打开）
   - 落地：使用中等高对比标题、弱化摘要与时间，行动按钮右下对齐。

4. **主次操作分层**
   - 核心动作常驻（如打开原文）；次级动作放在顶部 icon 区或次按钮。
   - 落地：刷新/同步放入顶部 icon button，避免底部连续大按钮。

5. **空态/错态可行动**
   - 主流产品空态不是“死文案”，而是“状态 + 下一步动作”。
   - 落地：统一引导卡片，包含标题、解释、主按钮、次按钮（如“查看信息源/重试”）。

## 本次改造策略

- 不改数据链路：继续 Room 本地优先 + source refresh 逻辑。
- 仅改 Feed 首屏 Compose 结构与表现层组件。
- 保留关键功能：源切换、刷新当前源、同步远端首页、打开原文。
