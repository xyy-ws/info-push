# Info Push Android 个人使用说明（v0.1.0-beta）

> 适合日常个人使用的最短指南。按下面步骤走，基本不会踩坑。

## 1. 第一次安装后先做什么

1. 打开 **信息源** 页面。
2. 先添加 1~2 个你常看的源（手动或 AI 搜索都可以）。
3. 回到 **首页**，切换到目标源，点 **刷新当前信息源**。
4. 看到列表有内容后，再开始收藏和阅读。

---

## 2. 核心功能怎么用

### 首页
- 用顶部信息源 chips 切换当前源。
- 点 **刷新当前信息源** 拉取该源最新内容。
- 点 **收藏/已收藏** 可收藏或取消收藏。
- 点 **打开原文** 会进入内置浏览器。

### 信息源
- 支持手动新增、AI 搜索添加、启用/禁用、删除、批量管理。
- 新增源采用“**先探测后入库**”：
  - 探测通过才会添加成功。
  - 探测失败会提示具体原因。
- 每个源会显示健康状态（成功/失败/暂无信息）。

### 收藏
- 支持实时过滤（输入即筛选）。
- 每条可直接 **打开原文** 或 **删除收藏**。

### 设置
- 支持数据 **导入/导出**（JSON）。
- 建议定期导出做备份。

---

## 3. 内置浏览器功能

打开文章后可使用：
- 返回 / 前进 / 刷新
- 外部浏览器打开
- 收藏当前文章
- 分享
- 复制链接

---

## 4. 常见提示是什么意思

- **该源拒绝访问(403)**：源站限制访问，换镜像或换地址。
- **该源暂无可解析内容**：源地址可访问但内容结构或当前数据不可用。
- **暂不支持该源格式**：当前解析器不支持该类型。
- **暂无信息（健康状态）**：该源还没执行过一次有效刷新。

---

## 5. 推荐个人使用习惯

1. 常用源控制在 5~15 个，避免噪音。
2. 每天固定 1~2 次手动刷新重点源。
3. 收藏后尽量当天清理（保留真正有价值内容）。
4. 每周导出一次 JSON 备份。

---

## 6. 当前版本已知限制

- 某些第三方 RSS（尤其部分 RSSHub）会受上游 403 限制。
- 健康状态是轻量实现，不是后台定时全量探针。
- 个别站点在内置浏览器首次加载仍可能有一次跳转等待。

---

## 7. 一句话工作流（推荐）

**信息源添加/检查 → 首页刷新浏览 → 有价值就收藏 → 收藏页二次筛选阅读 → 周期性导出备份**
